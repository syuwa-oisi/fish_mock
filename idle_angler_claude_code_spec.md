# Idle Angler — Claude Code 実装指示書

> **目的**: 既存の HTML モック（放置釣り・釣果4分岐経済）を、Tauri 2 を使った
> **画面四隅に常駐できる小型ウィジェット型**のネイティブデスクトップアプリに仕立て直し、
> 仲間に見せる「一通り遊べるデモ」に仕上げる。Steam 配布を見据えたネイティブ形式。
> （../td_mock の wall_siege_claude_code_spec.md と同じ進め方）

---

## 0. 前提・ゴール（最初に必ず読むこと）

- これは**社内デモ用の最短完成形**。凝った設計よりも「起動して眺めて遊べる」ことを最優先する。
- 既存モック（`idle_angler_mock.html`）は**土台として活用**する。ゼロから書き直さず、ロジックを移植・整理して拡張する。
- 技術スタックは **Tauri 2 + Vanilla TypeScript（フレームワークなし）**。
- ゲーム本体は **HTML5 Canvas + TypeScript**。Rust 側はウィンドウ制御（最前面・スナップ・トレイ）と
  セーブ保存先の取得など最小限に留める。
- **RMTは実装しない。** ただし「外付けすれば成立する」ことをコードで示す（→ 5章）。
- **完成の定義**: 下記「8. 完成チェックリスト」を全て満たし、`npm run tauri dev` で
  四隅常駐ウィジェットとして一通り遊べ、`npm run tauri build` でインストーラが生成できること。

## 2. 既存モックの扱い（移植方針）

`idle_angler_mock.html` を次のように分解して移植する:

- **`<style>`** → `src/styles.css`。CSS 変数はそのまま活かす。背景の「作業画面イメージ」(.desk) は**移植しない**（実機では本物のデスクトップがその役割）。
- **DOM 構造** → `index.html`。ウィジェット部分（#widget）のみ。
- **`<script>`** → TypeScript ファイル群に分割（下記3章）。まず素直に型を付けて動かす。
- 既存挙動（釣りループ・4分岐・餌Tierゲート・飲み込み装備・水槽/図鑑バフ・放流バフ・
  天候/時間帯・市場シミュ・オフライン進行・倍速）は**そのまま温存**する。

## 4. 実装する機能（このデモのスコープ）

### 4-1. 既存からの温存（必須・回帰させない）
- 放置釣りループ（竿Lv・装備・水槽・放流バフ・天候で速度/レア率が変動）。
- 魚の4分岐: 餌（Tier変換・格上げ連鎖）/ 捌く（現金＋飲み込み装備5/15/35/70%）/
  飾る（水槽恒久バフ）/ 放流（一時バフ）。
- 餌Tierゲート: E魚は小魚餌以上、L魚は活き餌以上、頂点魚「深淵の主」は特上餌のみ。
- 条件魚: ヌシ・オオイワナ=雨限定、タチウオ/幻影シードラゴン=夜限定。
- 釣り場3種の解放進行、図鑑（コンプでレア率+5%）、市場（需給ドリフト＋売買で価格変動）。
- オフライン進行（最大8h/400匹）と自動餌化トグル、デモ用倍速（×1/×4/×20）。

### 4-2. 常駐ウィジェット化（拡張・本命）
- **ボーダレス＝常駐するのは釣り人と水面の情景のみ**。ウィンドウ枠・パネルは出さない。
  - `transparent: true` + `decorations: false` + 透過Canvas（モックの描画をそのまま移植）で、
    デスクトップに情景だけが浮いて見える状態にする。
- 操作UIは**ホバーで出現する3ボタン**（×倍速 / 📌配置 / ☰メニュー）と、
  **☰からのポップアップ**（魚箱・装備・水槽図鑑・市場・設定）に集約する（モックと同じ構成）。
  - ポップアップ表示時はウィンドウを一時的に拡張（`setSize`）するか、
    子ウィンドウ（`WebviewWindow`）で出す。**デモではウィンドウ常時「情景+ポップアップ最大量」サイズの
    透過ウィンドウにして、クリック判定だけ情景・UI領域に限定する**のが最も簡単（Tauri の
    `setIgnoreCursorEvents` の領域制御が難しければ常時拡張で妥協してよい）。
- `alwaysOnTop: true`、`skipTaskbar: true` + **システムトレイ**（表示/終了）。
- **四隅スナップ**: モックの📌ボタン・設定の配置ボタンを Rust コマンド `snap_corner(corner)` に
  置き換え、モニタのワークエリア四隅へウィンドウを移動する。桟橋部分のドラッグ移動も可
  （`data-tauri-drag-region` 相当の領域をCanvas上部に設ける）。

### 4-3. セーブ（必須）
- localStorage から **アプリデータディレクトリの JSON** に移行（`tauri-plugin-store` か Rust コマンド）。
- オフライン進行は既存ロジックを流用。起動時トースト「💤 留守中に N 匹」。

### 4-4. 演出の底上げ（あれば加点）
- レジェンド/頂点魚ヒット時の専用演出（ライン緊張・スローモーション・画面フラッシュ）。
- 水槽タブで飾った魚が泳ぐミニアクアリウム描画。
- 効果音（任意・OFFデフォルト）。

## 5. マーケット外付け設計（このデモの核心・必読）

**RMT/Steam Market 連携は実装しない。** 代わりに「外付けできる形」をコードで証明する:

```ts
// market/api.ts — ゲーム本体はこのインターフェースしか知らない
export interface MarketAPI {
  quotes(): Promise<Quotes>;            // カテゴリ別需要係数
  listings(): Promise<Listing[]>;       // 出品一覧
  buy(id: ListingId): Promise<TradeResult>;
  sellBait(tier: number, qty: number): Promise<TradeResult>;
  sellEquip(item: Equip): Promise<TradeResult>;
  sellFish(fish: Fish): Promise<TradeResult>;
}
```

- 実装1: `local-sim.ts` … モックの `createLocalMarketSim()` を**非同期化して**移植（デモはこれで動かす）。
- 実装2: `steam-market.ts` … **スタブのみ作成**。`throw new Error('TODO: Steamworks 連携')` で良い。
  README に「ここを実装すれば外部経済に接続できる」と明記する。
- ゲーム本体・UIは `MarketAPI` 経由でのみ市場に触れること（同期呼び出しを残さない）。
- 価格・需要の発生源（餌連鎖の上位餌需要、図鑑コンプ需要、装備個体差）はゲームデザイン側に
  既に埋め込まれているので、アダプタを差し替えても経済が成立する——というのがデモの主張。

## 6. ウィンドウ・常駐まわり（tauri.conf.json 方針)

- 基本サイズ: 情景 408×190（ポップアップ分の余白を含めるなら 420×520 程度の透過ウィンドウ）。`resizable: false`。
- `alwaysOnTop: true`, `decorations: false`, `skipTaskbar: true`, **`transparent: true`（ボーダレスの要）**。
  - Windows では透過＋クリック領域の扱いに注意。問題が出たら「情景部分だけの小窓＋ポップアップは子ウィンドウ」構成に切り替える。
- トレイアイコン必須（仮アイコンで可）。トレイメニュー: 表示/非表示・終了。
- 「タスクバー埋め込み」等のOS依存の深掘りはしない。**透過ボーダレス常時最前面＋四隅スナップ＋トレイ**で意図を表現する。

## 9. 作業の進め方（推奨ステップ）

1. **足場**: create-tauri-app（vanilla-ts）→ 空ウィンドウ確認。
2. **移植**: モックの CSS/HTML/JS を `src/` に移し、型を付けて**既存機能がそのまま動く**状態に（ここで一度コミット）。
3. **市場分離**: `MarketAPI` インターフェース化＋local-sim 非同期化＋steam-market スタブ。
4. **常駐化**: 透過ウィンドウ設定・トレイ・四隅スナップ・ポップアップのウィンドウ連動。
5. **セーブ移行**: localStorage → アプリデータJSON。
6. **演出**: レジェンド演出・水槽アクアリウム（任意）。
7. **仕上げ**: README・型チェック・`tauri build`。

> 各ステップ終わりに動作確認とコミット。**動く状態を常に維持**しながら積み増す。

## 10. 設計上の注意・非ゴール

- **RMT・外部通信は実装しない**。アダプタの「形」を見せるのが目的。
- 過剰な抽象化・状態管理ライブラリ導入は不要。素の TS とモジュール分割で十分。
- バランス数値は仮で良いが、「餌の格上げ→高レア」「水槽/放流の使い分け」が体感できる程度には保つ。
- 見た目（ダーク基調・DotGothic16/M PLUS Rounded・ドット絵風シーン）は踏襲する。

## 付録: 既存モックの主要パラメータ（移植時の参照用）

- Canvas: 408×190（透過）。水面ブロブ y=104〜174（角丸24px）。釣り人 x≈350、ウキ x=236。
- 基本釣り間隔 6s ÷ 速度倍率。竿 Lv1〜8（+12%/Lv、費用 100×2^(Lv-1)）。
- レアリティ抽選基礎重み: C70 / R22 / E7 / L1.4（種数で按分、レア率バフと餌Tier超過で増幅）。
- 餌Tier: 0練り餌(∞) / 1小魚餌(C魚) / 2活き餌(R魚) / 3特上餌(E,L魚)。E魚はTier1+、L魚はTier2+、深淵の主はTier3。
- 飲み込み装備率: C5% / R15% / E35% / L70%。装備3種: リール(速度+8〜22%) / ルアー(レア+6〜18%) / お守り(サイズ+6〜16%・売値+5〜17%)、レアリティ係数 ×1 / ×1.6 / ×2.4 / ×4。
- 水槽バフ(×サイズ): C速度+4% / Rレア+5% / E速度+8%レア+5% / L速度+10%レア+12%。図鑑コンプ: 釣り場ごとにレア率+5%。
- 放流バフ: C速度+20%60s / Rレア+25%60s / E両方+30%90s / L両方+50%120s。
- 天候: 晴55%/雨30%/霧15%（300秒周期で再抽選）。雨: レア+10%、霧: サイズ+10%。時間帯: 朝昼夕夜 各75秒。
- 市場: 需要係数 0.6〜1.8、8秒ごとドリフト±4%、購入+0.06/売却-0.05、手数料10%、出品再生成60秒。
- 魚箱上限30（溢れは基礎値60%で自動買取）。オフライン進行: 最大8時間・400匹。

（数値はデモ用の叩き台。移植後に調整してよい。）
